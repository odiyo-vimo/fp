HMRC Integration Catalogue Reference
Source: Boomi Documentation (help.boomi.com) · June 2026
HMRC Integration Note: For the HMRC Boomi estate, the primary OOTB connectors in active use are Web Services Server (listener), HTTP Client / REST Client (outbound), Web Services SOAP Client (legacy SOAP bridging), Boomi Event Streams (Produce/Subscribe), Database, and SFTP v2. Partner and deprecated connectors are listed for completeness.
Technology Connectors
Protocol-level connectors — data format is independent of the connector. Use these for HTTP, FTP, SFTP, JMS, database, messaging, and similar transport patterns.
🌐
HTTP Client
Send and receive data over HTTP/HTTPS. Stateless — each document is an independent request. Generic; use REST Client or WSS for structured APIs.
OutboundHTTP/HTTPS⭐ HMRC Core
🔌
REST Client
Connect to any RESTful API. Handles OAuth, timeout management, response headers. Preferred over HTTP Client for structured REST integrations.
OutboundREST⭐ HMRC Core
📡
Web Services Server (WSS)
Listen for REST, SOAP, and HTTP requests in real-time. Primary inbound listener for HMRC-facing API endpoints. Used in P-101/P-105 patterns.
Inbound ListenerREST · SOAP⭐ HMRC Core
🔗
Web Services SOAP Client
Invoke SOAP 1.1/1.2 web services via WSDL. Used to bridge ChRIS/WebMethods legacy SOAP services. Key connector in HMRC legacy modernisation.
OutboundSOAP 1.1/1.2⭐ HMRC Core
🗄️
Database
Generic JDBC-based connector. Supports INSERT, GET, UPDATE, UPSERT, DELETE, stored procedures. Powers P-118 (DB as REST) and P-119 (governed DB access).
JDBCSQL⭐ HMRC Core
📁
SFTP V2
Secure file transfer over SSH. Supports create, download, delete, list with advanced JSch security configuration. Recommended over SFTP (Legacy).
File TransferSSH/SFTP
📁
SFTP (Legacy) Legacy
Original SFTP connector. Migrate to SFTP V2 for new integrations. Retained for backward compatibility with existing deployed processes.
LegacySSH/SFTP
📂
FTP V2
FTP file exchange with passive/active modes. Supports resume on interrupted transfers, symbolic link control, and metadata filtering.
File TransferFTP
📂
FTP (Legacy) Legacy
Original FTP connector. Superseded by FTP V2. Retained for deployed processes only.
LegacyFTP
💾
Disk V2
Read/write files on the Atom host filesystem. Useful for local staging, archive, and batch file processing scenarios.
Local FS
💾
Disk (Legacy) Legacy
Original Disk connector. Use Disk V2 for new implementations.
Legacy
📨
JMS (Java Message Service)
Send/receive messages via JMS-compliant brokers (ActiveMQ, IBM MQ, TIBCO, etc.). Supports queues and topics. Key for async messaging patterns.
MessagingJMS
⚡
Boomi Event Streams (Produce + Listen)
Native Kafka-backed broker within Boomi. Produce connector has no broker-side retry — build explicit Try/Catch. Powers P-120/P-121 patterns. Clustered Atom aware.
Kafka-backedPub/Sub⭐ HMRC Core
📬
AS2
EDI-over-HTTP file exchange using AS2 protocol with MDN acknowledgement. Used in B2B/Trading Partner scenarios.
B2B/EDIAS2
📬
AS4
Web-services-based secure messaging standard. ebMS 3.0 compliant. Successor to AS2 for government/regulated B2B messaging.
B2B/EDIebMS 3.0
🏥
MLLP Client / MLLP Server
Minimal Lower Layer Protocol for HL7 healthcare message transmission. Client sends; Server listens and initiates processes.
HealthcareHL7
✉️
Mail (IMAP / POP3 / SMTP)
Send and receive email. IMAP/POP3 for reading inbound mail; SMTP for sending. Supports attachments and triggered processing from inbox.
EmailSMTP/IMAP
☁️
Microsoft Azure Service Bus
Send/receive messages via Azure Service Bus queues and topics. Supports dead-letter handling and session-aware messaging.
AzureMessaging
☁️
Amazon SQS
Interact with AWS Simple Queue Service — send, receive, and delete messages from SQS queues. Standard and FIFO queues supported.
AWSMessaging
📢
Amazon SNS
Publish messages to AWS Simple Notification Service topics for fan-out delivery to SQS, Lambda, HTTP, and email endpoints.
AWSPub/Sub
🗂️
Boomi Message Queue Connector
Internal Boomi queue connector backed by the Atom's internal JMS broker. Used for intra-platform async handoffs within a single Atom/Molecule.
Internal Queue⭐ HMRC Core
🔑
LDAP
Query and update LDAP/Active Directory. Supports bind, search, add, modify, delete operations. Useful for identity/authorisation lookups.

Execute OS-level commands or shell scripts on the Atom host. Used for local scripting, batch invocation, or native CLI integrations.
