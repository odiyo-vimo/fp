Intelligrator — UI Screens (Standard User)

Interactive mockups of the three Standard-User screens, reproduced from the hand-drawn wireframes.

Boomi iPaaS · native MFT · config-driven provisioning — one MFT engine process, each approved interface becomes one config row (never a new component).

How to use this page

  1. Review the mockups below. Screens ①–③ are live HTML — the tiles, buttons and status indicators behave as they would in the product, so you can click through the interactions.
  2. Edit the source. Use the top bar to download intelligrator-ui-screens.drawio. Open it at app.diagrams.net or draw.io Desktop. To hand it to a Visio user, open the file in draw.io and choose Extras → Export as → Visio (.vsdx).
  3. Read the field-level spec. The Word document lists every field, its behaviour and validation, screen by screen — the build reference for the development team.
  4. Grab everything. docs.zip bundles the draw.io file and the Word spec in one download.

Three screens make up the Standard-User journey: a Dashboard to see submitted requests and launch actions, a guided Configure New File Transfer wizard to raise a request, and a Catalogue / History view to browse provisioned transfers and trigger ad-hoc runs. Sample data is illustrative only.
